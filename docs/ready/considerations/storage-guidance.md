# <a name="storage-design-decisions"></a>Decisões de design de armazenamento

Os recursos de armazenamento são essenciais para dar suporte a cargas de trabalho e serviços hospedados na nuvem. Como parte de suas preparações de preparação para adoção de nuvem, leia este artigo para ajudá-lo a planejar e atender às suas necessidades de armazenamento.

## <a name="select-storage-tools-and-services-to-support-your-workloads"></a>Selecione Ferramentas e serviços de armazenamento para dar suporte às suas cargas de trabalho

O [armazenamento do Azure](https://docs.microsoft.com/azure/storage) é o serviço gerenciado da plataforma Azure para fornecer armazenamento em nuvem. O armazenamento do Azure é composto por vários serviços principais e recursos de suporte. O armazenamento no Azure é altamente disponível, seguro, durável, escalonável e redundante. Examine os cenários e considerações descritos aqui para escolher os serviços do Azure relevantes e as arquiteturas corretas para atender aos requisitos de carga de trabalho, governança e armazenamento de dados da sua organização.

### <a name="key-questions"></a>Principais perguntas

Responda às seguintes perguntas sobre suas cargas de trabalho para ajudá-lo a tomar decisões com base na árvore de decisão do armazenamento do Azure:

- **Suas cargas de trabalho exigem armazenamento em disco para dar suporte à implantação de máquinas virtuais de IaaS (infraestrutura como serviço)?** [Armazenamento em disco do Azure](https://docs.microsoft.com/azure/virtual-machines/windows/managed-disks-overview) fornece recursos de disco virtual para máquinas virtuais IaaS.
- **Será necessário fornecer imagens, documentos ou outras mídias para download como parte de suas cargas de trabalho?** O [armazenamento de BLOBs do Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-introduction) fornece a capacidade de [hospedar arquivos estáticos](https://docs.microsoft.com/azure/storage/blobs/storage-blob-static-website), que são então acessíveis para download pela Internet. Você pode tornar ativos hospedados no armazenamento de BLOBs públicos ou pode [limitar ativos a usuários autorizados](https://docs.microsoft.com/azure/storage/common/storage-auth) por meio do Azure Active Directory (Azure AD), chaves compartilhadas ou assinaturas de acesso compartilhado.
- **Será necessário um local para armazenar logs de máquina virtual, logs de aplicativo e dados de análise?** Você pode usar o armazenamento de BLOBs do Azure para [armazenar Azure monitor dados de log](https://docs.microsoft.com/azure/storage/common/storage-analytics).
- **Será necessário fornecer um local para backup, recuperação de desastre ou arquivamento de dados relacionados à carga de trabalho?** Armazenamento em Disco do Azure usa o armazenamento de BLOBs do Azure para fornecer [recursos de backup e recuperação de desastre](https://docs.microsoft.com/azure/virtual-machines/windows/backup-and-disaster-recovery-for-azure-iaas-disks). Você também pode usar o armazenamento de BLOBs como um local para fazer backup de outros recursos, como [dados de SQL Server](https://docs.microsoft.com/sql/relational-databases/backup-restore/sql-server-backup-and-restore-with-microsoft-azure-blob-storage-service?view=sql-server-2017)hospedados por VM locais ou IaaS.
- **Será necessário dar suporte a cargas de trabalho de análise de Big Data?** O [Azure data Lake Storage Gen 2](https://docs.microsoft.com/azure/storage/blobs/data-lake-storage-introduction) é criado sobre o armazenamento de BLOBs do Azure. O Data Lake Storage Gen 2 pode dar suporte à funcionalidade data Lake de grande porte. Ele também pode lidar com o armazenamento de petabytes de informações e, ao mesmo tempo, manter centenas de gigabits de taxa de transferência.
- **Será necessário fornecer compartilhamentos de arquivos nativos de nuvem?** O Azure tem dois serviços principais que fornecem compartilhamentos de arquivos hospedados na nuvem: Azure NetApp Files e arquivos do Azure. O [Azure NetApp files](https://docs.microsoft.com/azure/azure-netapp-files/azure-netapp-files-introduction) fornece compartilhamentos NFS de alto desempenho que são adequados para cargas de trabalho corporativas comuns como SAP. [Os arquivos do Azure](https://docs.microsoft.com/azure/storage/files/storage-files-introduction) fornecem compartilhamentos de arquivos acessíveis via SMB 3,0 e HTTPS.
- **Você precisará oferecer suporte ao armazenamento de nuvem híbrida para cargas de trabalho de HPC (computação de alto desempenho) local?** O [avere vFXT for Azure](https://docs.microsoft.com/azure/avere-vfxt/avere-vfxt-overview) é uma solução de cache híbrido que você pode usar para expandir seus recursos de armazenamento local usando o armazenamento baseado em nuvem. O avere vFXT for Azure é otimizado para cargas de trabalho HPC de leitura pesada que envolvem farms de computação de 1.000 a 40.000 núcleos de CPU. O avere vFXT para Azure pode integrar o NAS (armazenamento conectado à rede) de hardware local, o armazenamento de BLOBs do Azure ou ambos.
- **Você precisará executar o arquivamento em larga escala e a sincronização de seus dados locais para a nuvem?** [Azure data Box](https://docs.microsoft.com/azure/databox-family) produtos foram projetados para ajudá-lo a mover grandes quantidades de dados do seu ambiente local para a nuvem. [Gateway do Azure data Box](https://docs.microsoft.com/azure/databox-online/data-box-gateway-overview) é um dispositivo virtual que reside no local. Gateway do Data Box ajuda a gerenciar a migração de dados em larga escala para a nuvem. Se precisar analisar, transformar ou filtrar dados antes de movê-los para a nuvem, você pode usar [Azure data Box Edge](https://docs.microsoft.com/azure/databox-online/data-box-edge-overview), um dispositivo de computação de borda física habilitado para ia que é implantado em seu ambiente local. Data Box Edge acelera o processamento e a transferência segura de dados para o Azure.
- **Deseja expandir um compartilhamento de arquivos local existente para usar o armazenamento em nuvem?** [Sincronização de arquivos do Azure](https://docs.microsoft.com/azure/storage/files/storage-sync-files-deployment-guide) permite que você use o serviço de arquivos do Azure como uma extensão de compartilhamentos de arquivos hospedados em suas máquinas locais do Windows Server. O serviço de sincronização transforma o Windows Server em um cache rápido do compartilhamento de arquivos do Azure. Ele permite que seus computadores locais que acessam o compartilhamento usem qualquer protocolo que esteja disponível no Windows Server.

Se eles precisarem acessar esses dados mais tarde, fica certo onde eles o colocam. |
| Tenho intermitências de quantidades de dados que aparecem de uma só vez. | [Azure Data Box Edge ou Gateway do Azure Data Box](https://docs.microsoft.com/azure/databox-online) | Gerencie grandes quantidades de dados que aparecem de uma só vez, como quando um carro autônomo puxa de volta para a garagem ou um computador de sequenciamento de gene conclui sua análise. Copie todos esses dados para Gateway do Data Box em velocidades locais rápidas e, em seguida, deixe o gateway carregá-lo conforme a sua rede permitir.

## <a name="learn-more-about-azure-storage-services"></a>Saiba mais sobre os serviços de armazenamento do Azure

Depois de identificar as ferramentas do Azure que melhor atendem aos seus requisitos, use a documentação detalhada vinculada na tabela a seguir para se familiarizar com esses serviços:

| **Serviço** | **Ndescrição** |
|---|---|
| [Armazenamento de Blobs do Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-introduction) | O armazenamento de BLOBs do Azure é a solução de armazenamento de objetos da Microsoft para a nuvem. O armazenamento de BLOBs é otimizado para armazenar grandes quantidades de dados não estruturados. Dados não estruturados são dados que não aderem a um modelo ou definição de dados específico, como texto ou dados binários.<br/><br/>O armazenamento de BLOBs foi projetado para:<ul><li>Servir imagens ou documentos diretamente para um navegador.</li><li>Armazenando arquivos para acesso distribuído.</li><li>Transmissão de vídeo e áudio.</li><li>Gravando em arquivos de log.</li><li>Armazenamento de dados para backup e restauração, recuperação de desastres e arquivamento.</li><li>Armazenamento de dados para análise por um serviço local ou hospedado no Azure.</li></ul> |

## <a name="data-redundancy-and-availability"></a>Redundância e disponibilidade de dados

O armazenamento do Azure tem várias opções de redundância para ajudar a garantir a durabilidade e a alta disponibilidade com base nas necessidades do cliente: LRS (armazenamento com redundância local), ZRS (armazenamento com redundância de zona), GRS (armazenamento com redundância geográfica) e armazenamento com redundância geográfica com acesso de leitura (RA-GRS) .

Consulte [redundância de armazenamento do Azure](https://docs.microsoft.com/azure/storage/common/storage-redundancy) para saber mais sobre esses recursos e como você pode decidir sobre a melhor opção de redundância para seus casos de uso. Além disso, os SLAs (contratos de nível de serviço) para serviços de armazenamento fornecem garantias que são apoiadas financeiramente. Para obter mais informações, consulte [SLA para Managed disks](https://azure.microsoft.com/support/legal/sla/managed-disks/v1_0), [SLA para máquinas virtuais](https://azure.microsoft.com/support/legal/sla/virtual-machines/v1_8)e [SLA para contas de armazenamento](https://azure.microsoft.com/support/legal/sla/storage/v1_4).

Para obter ajuda sobre como planejar a solução certa para discos do Azure, consulte [backup e recuperação de desastre para armazenamento em disco do Azure](https://docs.microsoft.com/azure/virtual-machines/windows/backup-and-disaster-recovery-for-azure-iaas-disks).

## <a name="security"></a>Segurança

Para ajudá-lo a proteger seus dados na nuvem, o armazenamento do Azure oferece várias práticas recomendadas para segurança de dados e criptografia para dados em repouso e em trânsito. Você pode:

- Proteja a conta de armazenamento usando o RBAC e o Azure AD.
- Proteja os dados em trânsito entre um aplicativo e o Azure usando criptografia do lado do cliente, HTTPS ou SMB 3,0.
- Defina os dados a serem criptografados automaticamente quando gravados no armazenamento do Azure usando a criptografia do serviço de armazenamento.
- Conceda acesso delegado aos objetos de dados no armazenamento do Azure usando assinaturas de acesso compartilhado.
- Use a análise para acompanhar o método de autenticação que alguém está usando ao acessar o armazenamento no Azure.

Esses recursos de segurança se aplicam ao armazenamento de BLOBs do Azure (bloco e página) e aos arquivos do Azure. Obtenha diretrizes de segurança de armazenamento detalhadas no [Guia de segurança do armazenamento do Azure](https://docs.microsoft.com/azure/storage/common/storage-security-guide).

A [criptografia do serviço de armazenamento](https://docs.microsoft.com/azure/storage/storage-service-encryption) fornece criptografia em repouso e protege seus dados para atender aos compromissos de segurança e conformidade de sua organização. A criptografia do serviço de armazenamento é habilitada por padrão para todos os discos gerenciados, instantâneos e imagens em todas as regiões do Azure. A partir de 10 de junho de 2017, todos os novos discos gerenciados, instantâneos, imagens e novos dados gravados em discos gerenciados existentes são automaticamente criptografados em repouso com chaves gerenciadas pela Microsoft. Visite as [perguntas frequentes para discos gerenciados](https://docs.microsoft.com/azure/virtual-machines/windows/faq-for-disks#managed-disks-and-storage-service-encryption) para obter mais detalhes.

Azure Disk Encryption permite criptografar discos gerenciados que são anexados a VMs IaaS como sistema operacional e discos de dados em repouso e em trânsito usando as chaves armazenadas no [Azure Key Vault](https://azure.microsoft.com/documentation/services/key-vault). Para o Windows, as unidades são criptografadas usando a tecnologia de criptografia [BitLocker](/windows/security/information-protection/bitlocker/bitlocker-overview) padrão do setor. Para o Linux, os discos são criptografados usando o subsistema [DM-cript](https://wikipedia.org/wiki/Dm-crypt) . O processo de criptografia é integrado com o Azure Key Vault para permitir que você controle e gerencie as chaves de criptografia de disco. Para obter mais informações, consulte [Azure Disk Encryption para VMs IaaS Windows e Linux](https://docs.microsoft.com/azure/security/azure-security-disk-encryption-overview).

## <a name="regional-availability"></a>Disponibilidade regional

Você pode usar o Azure para entregar serviços na escala de que precisa para acessar seus clientes e parceiros *onde quer que estejam*. As páginas de disponibilidade regional dos [discos gerenciados](https://azure.microsoft.com/global-infrastructure/services/?products=managed-disks) e [do armazenamento do Azure](https://azure.microsoft.com/global-infrastructure/services/?products=storage) mostram as regiões em que esses serviços estão disponíveis. Verificar a disponibilidade regional de um serviço com antecedência pode ajudá-lo a tomar a decisão certa para sua carga de trabalho e necessidades do cliente.

Os Managed disks estão disponíveis em todas as regiões do Azure que têm SSD Premium e SSD Standard ofertas. Embora SSD Ultra atualmente esteja em visualização pública, ela é oferecida em apenas uma zona de disponibilidade, a região leste dos EUA 2. Verifique a disponibilidade regional ao planejar cargas de trabalho críticas e de camada superior que exigem SSD Ultra.

O armazenamento de BLOBs quente e frio, Data Lake Storage Gen2 e armazenamento de arquivos do Azure estão disponíveis em todas as regiões do Azure. O armazenamento de Bob de arquivamento, os compartilhamentos de arquivos Premium e o armazenamento de blobs de blocos Premium são limitados a determinadas regiões. Recomendamos que você consulte a página regiões para verificar o status mais recente da disponibilidade regional.

Para saber mais sobre a infraestrutura global do Azure, confira a [página regiões do Azure](https://azure.microsoft.com/global-infrastructure/regions). Você também pode consultar a página [produtos disponíveis por região](https://azure.microsoft.com/global-infrastructure/services/?products=storage) para obter detalhes específicos sobre o que está disponível em cada região do Azure.

## <a name="data-residency-and-compliance-requirements"></a>Requisitos de conformidade e residência de dados

Os requisitos legais e contratuais relacionados ao armazenamento de dados geralmente serão aplicados às suas cargas de trabalho. Esses requisitos podem variar de acordo com o local da sua organização, a jurisdição dos ativos físicos que hospedam seus armazenamentos de dados e seu setor de negócios aplicável. Os componentes das obrigações de dados a serem considerados incluem a classificação de dados, o local dos dados e as respectivas responsabilidades de proteção de dados no modelo de responsabilidade compartilhada. Para obter ajuda com a compreensão desses requisitos, consulte a white paper [obter segurança e residência de dados em conformidade com o Azure](https://azure.microsoft.com/resources/achieving-compliant-data-residency-and-security-with-azure).

Parte dos seus esforços de conformidade pode incluir o controle de onde os recursos do banco de dados estão localizados fisicamente. As regiões do Azure são organizadas em grupos chamados geografias. Uma [Geografia do Azure](https://azure.microsoft.com/global-infrastructure/geographies) garante que os requisitos de residência de dados, soberania, conformidade e resiliência sejam respeitados em limites geográficos e políticos. Se suas cargas de trabalho estiverem sujeitas à soberania de dados ou outros requisitos de conformidade, você deverá implantar seus recursos de armazenamento em regiões que estejam em uma geografia do Azure em conformidade.
